## 修复 BookmarkCard 菜单过渡动画不生效的问题

### 问题分析
在 `BookmarkCard.jsx` 中使用 `{showActions && (...)}` 条件渲染菜单，当 `showActions` 从 false 变为 true 时，整个菜单元素被添加到 DOM 中，导致 CSS 过渡动画无法生效。

### 解决方案

**修改 BookmarkCard.jsx**：
- 将条件渲染 `{showActions && (...)}` 改为始终渲染菜单元素
- 使用 CSS 的 `visibility`、`opacity` 和 `pointer-events` 控制菜单的显示/隐藏状态
- 添加 `actions-open` 类来触发动画效果

**修改 BookmarkCard.scss**：
- 保持现有的过渡动画样式不变
- 确保移动端菜单的动画效果正常工作

### 具体代码修改

**BookmarkCard.jsx**（第82-90行）：
```jsx
// 修改前
{showActions && (
  <div className="bookmark-actions-overlay" onClick={(e) => e.stopPropagation()}>
    ...
  </div>
)}

// 修改后
<div className={`bookmark-actions-overlay ${showActions ? 'visible' : ''}`} onClick={(e) => e.stopPropagation()}>
  ...
</div>
```

**BookmarkCard.scss**（第196-213行）：
```scss
.bookmark-actions-overlay {
  // 保持原有样式
  opacity: 0;
  visibility: hidden;
  transform: translateY(-8px) scale(0.95);
  transition: opacity 0.2s ease, transform 0.2s ease, visibility 0.2s;
  
  &.visible {
    opacity: 1;
    visibility: visible;
    transform: translateY(0) scale(1);
    pointer-events: auto;
  }
}
```

这样修改后，菜单元素始终存在于 DOM 中，通过 CSS 类控制显示状态，过渡动画就能正常工作了。
