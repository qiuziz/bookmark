
# 一、精准生成提示词

创建一个书签管理工具项目，使用React+TypeScript+Vite，具备以下功能：
1. 核心功能：书签增删改查、置顶、导入导出、文件夹管理
2. 壁纸功能：集成Bing API随机获取壁纸，启动自动设置
3. 视觉设计：
   - 卡片和头部应用高斯模糊效果(backdrop-filter: blur)
   - 卡片背景透明度40%，悬停时60%
   - 响应式布局：小屏3列/中屏4列/大屏5列/超大屏6列
4. 技术配置：
   - 支持双模式构建(普通模式/插件模式)
   - 使用pnpm作为包管理器
   - 配置GitHub Action自动发布release
5. 特殊要求：
   - 自定义图标功能





# 项目完整描述
## 1. 项目初始想法
创建一个现代化的书签管理工具，解决浏览器默认书签管理的不足，提供更美观、更实用的书签管理体验。初始功能规划包括书签的增删改查、导入导出、置顶功能和移动端菜单支持，使用TypeScript重构项目结构以提高代码质量。

## 2. 迭代过程
### 2.1 基础功能实现
- 完成了书签管理的核心功能
- 实现了响应式布局
- 使用TypeScript重构了项目结构
### 2.2 视觉设计优化
- 增加了壁纸功能，最初使用本地图片，后改为使用Bing API获取随机壁纸
### 2.3 需求调整与优化
- 根据用户反馈，调整了磨砂效果的应用范围：仅卡片和头部应用高斯模糊，背景图保持清晰
- 重新设计了UI，特别是优化了书签卡片的视觉效果
- 调整了卡片透明度：默认40%，悬停时60%
- 减小了书签卡片宽度，增加了响应式列数，使界面更紧凑
### 2.4 功能增强
- 实现了启动自动设置壁纸功能
- 增加了从Chrome收藏夹导入书签的功能，自动过滤系统文件夹
- 添加了自定义图标URL输入功能
- 优化了图标加载状态显示
### 2.5 项目管理优化
- 配置了GitHub Action自动构建和发布release的工作流
- 完善了CHANGELOG.md，添加了详细的版本说明
## 3. 最终成果
### 3.1 核心功能
- 完整的书签管理功能：增删改查、置顶、导入导出
- 文件夹管理功能
- 壁纸功能：支持Bing API获取随机壁纸，启动自动设置
- 自定义图标功能
### 3.2 视觉效果
- 卡片和头部应用高斯模糊效果，背景图清晰可见
- 卡片背景透明度40%，悬停时60%
- 响应式布局：小屏幕(≤600px)3列，中等屏幕(600-900px)4列，大屏幕(900-1200px)5列，超大屏幕(>1200px)6列
- 卡片悬停效果：上浮、缩放、阴影加深
### 3.3 技术实现
- 使用React+TypeScript+Vite构建
- 支持双模式构建：普通模式和插件模式
- 使用pnpm作为包管理器
- 配置了GitHub Action自动发布release
- 优化了代码结构和性能
### 3.4 项目管理
- 清晰的版本控制和CHANGELOG
- 自动化的构建和发布流程
## 4. 使用方式
### 4.1 开发模式
```
pnpm install
pnpm run dev
```
### 4.2 构建普通版本
```
pnpm run build
```
### 4.3 构建插件版本
```
pnpm run build:plugin
```
### 4.4 发布release
1. 更新package.json中的版本号
2. 创建并推送标签： git tag vx.x.x && git push origin vx.x.x
3. GitHub Action会自动构建并发布release
